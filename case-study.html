Pizza Store Database Case Study

A 3NF-compliant database system with PHP/MySQL integration for order management.

Project Overview

I designed and developed a fully functional database system for a pizza store to efficiently manage orders and customer data. The core deliverable was a 3NF-compliant relational database built with MySQL, paired with a PHP front-end for data entry and management through user-friendly forms.

Goals

  • Design a normalized database schema that minimizes data redundancy and ensures data integrity by achieving Third Normal Form (3NF).
  • Create a front-end interface using PHP for managing pizzas, customers, orders, and order details.
  • Implement robust data validation and relationship management without relying on auto-increment keys.
  • Support essential pizza store operations: placing orders, updating menus, tracking customers, and reviewing order histories.

Database Design

Entities & Relationships

  • Customers: Stores customer details with a unique customer ID.
  • Pizzas: Contains pizza types, sizes, and prices.
  • Orders: Represents each order placed by a customer with an order ID, date/time, and status.
  • Order_Items: Line items detailing which pizzas and quantities are included in each order.

Normalization

  • Each table was designed to eliminate duplicate data and ensure atomicity of fields.
  • The schema adheres strictly to 3NF by:
    • Ensuring no partial dependencies on composite keys.
    • Eliminating transitive dependencies to maintain data integrity.
  • Primary keys were manually assigned, avoiding auto-increment, to satisfy project constraints.

Front-End Implementation

Forms and PHP

  • Created PHP forms for adding, updating, and deleting records across all tables.
  • Included input validation to prevent invalid or incomplete data from entering the system.
  • Used dropdowns and input fields for smooth selection of pizza types and customer IDs.
  • Used prepared statements to enhance database security and prevent SQL injection.

User Experience

The interface was kept simple and intuitive, with clear visual feedback for success or error states. The focus was on functionality, accessibility, and reducing friction in the ordering workflow.

Challenges & Solutions

  • Maintaining 3NF without auto-increment: Manually managed keys by tracking max IDs and incrementing them programmatically.
  • Validation: Used both client-side (HTML5) and server-side (PHP) validation to maintain data integrity.
  • Form usability: Designed for clarity and error-prevention through iterative user testing and refinement.

Results & Learnings

  • Gained experience designing normalized databases that meet real-world standards for efficiency and integrity.
  • Built a working CRUD interface integrated with MySQL and PHP logic.
  • Deepened my understanding of security, validation, and relational data modeling.

The Pizza Store Database is now a scalable, maintainable system capable of supporting daily pizza ordering operations with minimal risk of data anomalies.

Future Enhancements

  • Integrate user authentication for employee roles and permissions.
  • Enhance interactivity with JavaScript for real-time form validation and dynamic price calculations.
